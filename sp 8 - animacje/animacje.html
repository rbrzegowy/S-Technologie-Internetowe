Animacje CSS

1

transition

  • Zmienia stan po interakcji
  • Może też płynnie wprowadzać element z display: none dzięki transition-behavior i @starting-style
.box {
  transition:   transform 0.45s ease,
  border-radius 0.45s ease;
}
.box:hover {
  transform: translateX(24px);
  border-radius: 50%;
}

.panel {
  display: none;
  opacity: 0;
  transition: opacity 0.35s ease, display 0.35s allow-discrete;
  transition-behavior: allow-discrete;
}
.show .panel {
  display: block;
  opacity: 1;
  @starting-style { opacity: 0; }
}
Wejście z display: none
2

@keyframes

  • Predefiniowana animacja
  • Animacja definiowana jest krokami (0-100%)
@keyframes spin {
  from { transform: rotate(0deg); }
  to { transform: rotate(360deg); }
}
.spinner {
  animation: spin 2s linear infinite;
}
3

timing-function

  • Timing function definiuje przebieg animacji w czasie
  • Zmiana timing-function wpływa na dynamikę ruchu
  • Funkcje czasowe zmieniają odczuwany czas trwania animacji
  • Do dyspozycji mamy wbudowane timing functions
  • Dodatkowo - cubic-bezier() pozwala definiować własne krzywe
.linear { animation-timing-function: linear; }
.ease { animation-timing-function: ease; }
.bezier { animation-timing-function: cubic-bezier(0.39, 1.33, 0.55, -0.23); }

4

delay

  • Delay opóźnia rozpoczęcie animacji
  • Przy tych samych keyframes i narastających opóźnieniach dostajemy efekt kontynuacji animacji po poprzedniej
.a { animation: pop 1.5s ease-out 0s infinite; }
.b { animation: pop 1.5s ease-out 0.35s infinite; }
.c { animation: pop 1.5s ease-out 0.7s infinite; }
5

fill-mode

  • forwards zatrzymuje ostatni stan animacji
  • backwards aplikuje styl 0% jeszcze przed startem (podczas delay)
.fill-forwards { animation: fill-slide 1.4s ease forwards; }
.fill-backwards { animation: fill-slide 1.4s ease 0.8s backwards; }
6

direction

  • normal, reverse, alternate, alternate-reverse
  • Ta sama animacja, inny kierunek odtwarzania
.dir-normal { animation-direction: normal; }
.dir-reverse { animation-direction: reverse; }
.dir-alt { animation-direction: alternate; }
.dir-alt-rev { animation-direction: alternate-reverse; }

7

timeline i range

  • Animacja może zależeć od scrolla zamiast czasu
  • animation-timeline i animation-range sterują zakresem aktywacji
.range-item {
  animation: reveal-range linear both;
  animation-timeline: view(block);
  animation-range: entry 10% cover 50%;
}

8

play-state

  • animation-play-state: paused zatrzymuje animację
  • Częsty usecase: pauza po najechaniu
.play-wheel { animation: spin 2.5s linear infinite; }
.play-wheel:hover { animation-play-state: paused; }

Najedź kursorem na koło, żeby zatrzymać obrót.

9

composition

  • animation-composition: add pozwala składać wiele animacji
  • Przykład łączy ruch pionowy i poziomy na jednym elemencie
  • Możliwe wartości: add, replace (default), accumulate
.composition-dot {
  animation: comp-x 2.2s ease-in-out infinite, comp-y 1.3s ease-in-out infinite alternate;
  animation-composition: add, add;
}
10

Debugowanie animacji w Google Chrome

  • Testuj wpływ właściwości bezpośrednio w DevTools, np. animation-duration i animation-timing-function
  • W panelu Animations zatrzymuj, spowalniaj i analizuj przebieg animacji
  • Użyj Rendering - Paint flashing, aby wykryć kosztowne repainty
  • Również Rendering - testuj reduced motion
/* szybki tryb debugowania */
.debug * {
  animation-play-state: paused !important;
  transition: none !important;
}
/* potem włączaj pojedyncze animacje i porównuj efekt */
11

Dobre praktyki

  • Pereferuj transform nad innymi formami przekształceń
  • Pamiętaj dla kogo tworzysz animacje (rodzaj strony/aplikacji, sposób jej wykorzystania)
  • Animacje UI - krótkie, 100-300 ms
  • Przejścia pomiędzy widokami - ok. 300-600ms
  • Wspieraj prefers-reduced-motion
  • Testuj klawiaturę - stan :focus-visible w parze z :hover (EAA i WCAG)
@media (prefers-reduced-motion: reduce) {
  * {
    animation: none !important;
    transition: none !important;
  }
}
