WCAG w pigułce - najważniejsze kryteria

  • Trzy poziomy zgodności: A (minimum), AA (standard branżowy i wymóg prawny) oraz AAA (najwyższy, trudny do spełnienia w całości).
  • Większość regulacji (EAA, WCAG 2.1) wymaga poziomu AA.
  • Kontrast tekstu: minimum w 4.5:1 dla zwykłego tekstu (AA) lub 7:1 (AAA).
  • Nie polegaj tylko na kolorze: błąd formularza musi mieć też opis lub ikonę.
  • Alt dla obrazów: opisuj obrazki merytoryczne, dekoracyjne zostaw jako alt="". Pisz nie tylko że na obrazku jest wykres, napisz co wynika z wykresu.
  • Hierarchia nagłówków to mapa strony dla czytnika ekranu, nie tylko styl.
img dekoracyjny: alt=""
img merytoryczny: alt="Wykres sprzedaży Q1-Q4, 30% wzrost"

.error { border-color: #b52a1d; }
<p class="error-text">Podaj poprawny e-mail</p>

Poziomy A / AA / AAA

Poziom Co obejmuje Kontrast tekstu
A Absolutne minimum: alt dla obrazów, etykiety formularzy, możliwość obsługi klawiaturą, brak migania ekranu, struktura dokumentu, kolejność fokusu i nawigacji brak wymogu
AA Standard prawny (EAA): powiększenie do 200%, podpisy wideo, widoczny focus (outline) 4,5 : 1
AAA Najwyższy poziom: rozszerzone opisy (np. audiodeskrypcja), większe dostępność treści (np. zoom, odstępy między znakami), bardziej szczegółowe opisy, brak limitu czasu (lub wydłużenie) 7 : 1

Accessible Rich Internet Applications

  • ARIA to zestaw atrybutów które pozwalają na lepsze opisanie elementów stron dla technologi asystujących (np. czytników ekranu)
  • Zasada główna - jeśli istnieje semantyczny element HTML, użyj go zamiast role ARIA.
  • ARIA dodajemy wtedy, gdy znacznik/atrybuty HTML nie pozwalają dostarczyć kontekstu informacji.
  • 1. Opisy i etykiety
    • aria-label - nadaje etykietę "z powietrza" (gdy brak widocznego tekstu).
    • aria-labelledby - wskazuje element, który opisuje dany element.
    • aria-describedby - dodatkowy opis (np. hint, error).
    • Użycie: ikony, inputy i custom buttony bez czytelnego tekstu widocznego na ekranie.
  • 2. Stan elementu
    • aria-checked - checkbox / toggle (true/false/mixed).
    • aria-selected - zaznaczenie w listach i tabach.
    • aria-expanded - informuje czy element jest rozwinięty (accordion, menu).
    • aria-pressed - stan przycisku typu toggle.
    • aria-disabled - nieaktywny element (czytelniejsze niż samo CSS).
  • 3. Widoczność i dostępność
    • aria-hidden="true" - ukrywa element przed screen readerem.
    • aria-live - ogłasza dynamiczne zmiany (np. alerty).
    • Wartości: polite, assertive.
  • 4. Relacje między elementami
    • aria-controls - wskazuje, co dany element kontroluje.
    • aria-owns - wirtualnie "przypisuje" element w DOM.
    • aria-activedescendant - wskazuje aktywny element w złożonych widgetach.
  • 5. Walidacja i formularze
    • aria-required - oznacza pole wymagane.
    • aria-invalid - sygnalizuje błąd w polu.
    • aria-errormessage - wskazuje element z komunikatem błędu.
  • No ARIA is better than bad ARIA: jeśli natywny HTML wystarcza, nie dodawaj ARIA na siłę.
<button aria-label="Zamknij panel">X</button>
<input id="email" aria-describedby="emailHint emailError" aria-invalid="true" />
<p id="emailHint">Użyj adresu firmowego.</p>
<p id="emailError" role="alert">Podaj poprawny e-mail.</p>

<button aria-expanded="false" aria-controls="faq1">Pokaż odpowiedź</button>
<div id="faq1" hidden>Treść odpowiedzi...</div>

Nawigacja klawiaturą i focus

  • Focus ring jest sygnałem orientacji. Usuwanie outline: none bez zamiennika to poważny błąd.
  • tabindex="0" dołącza element do naturalnej kolejności tabulacji.
  • tabindex="-1" pozwala ustawić focus programowo (np. po otwarciu modala).
  • Użytkownik nie może utknąć w komponencie. Trzeba zapewnić wejście i wyjście.

Szybki audyt - jak testować?

  • Narzędzia automatyczne (Lighthouse, WAVE, axe DevTools) są pomocne, ale wykrywają tylko część problemów.
  • Test "No Mouse": przejdź cały flow tylko klawiszami Tab, Shift+Tab, Enter.
  • Czytniki ekranu: NVDA (Windows) i VoiceOver (macOS) powinny czytać logiczną, spójną historię strony.
  • Jeden krótki film z realnego użycia czytnika zwykle zmienia perspektywę szybciej niż dziesiątki slajdów.
