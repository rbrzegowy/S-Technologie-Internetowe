Zmienne w CSS

1

Podstawy zmiennych

  • Zmienne zawsze definiujemy w jakimeś kontekście
  • Najczęstszy "first choice" - globalnie w selektorze :root (html)
  • Zmienne z :root są dostępne w całym drzewie CSS
  • Zmienne mogą przechowywać pojedyncze wartości lub całe wyrażenia (np. dla box-shadow)
  • Definiowanie zmiennej: --nazwa-zmiennej: wartość
  • Używanie: var(--nazwa-zmiennej, fallback)
:root {
--accent: #ec5a2f;
--space: 1rem;
--shadow-pop: 0 22px 44px rgba(34, 48, 71, 0.18);
}
.card {
padding: var(--space);
}

Nadpisywanie i lokalny scope zmiennych

  • Można deklarować zmienne lokalnie - idea komponentu
  • Możliwe jest nadpisywanie zmiennych globalnych lokalnie
  • Przykładowe scenariusze: komponenty, różne warianty elementu, nadpisywanie wartości w @media
.card--success {
--card-accent: #2f744d;
}
.card {
border-color: var(--card-accent);
}

3

Fallback w var()

  • Jeśli zmienna nie istnieje, można podać wartość domyślną
  • Składnia: var(--nazwa, fallback)
  • Fallback może być również zmienną: var(--fallback, var(--def-value))
color:
var(--badge-ink, #223047);
background: var(--badge-bg, #ffe7dd);

Przełączanie motywu ciemny/jasny

  • W :root definiujemy zmienne dla domyślnego trybu
  • Zmienne są nadpisywane w selektorze definiującym tryb (np. .dark lub [data-theme]="dark").
  • Zmienne świetnie pracują z funkcją light-dark()
  • ...oraz @media (prefers-color-scheme: dark/light)
.theme-panel[data-theme='dark'] {
--surface: #1d2737;
--ink-local: #f3f7ff; }

Zmienne jako baza biblioteki/motywu CSS

  • Najlepiej traktować je jak mały design system
  • Osobno trzymaj kolory, typografię, layout, animacje itd.
  • Wtedy łatwo skalować projekt i utrzymać spójność

Budowanie motywu kolorystycznego

  • Przykład: Material Design
  • Zacznij od 2-4 kolorów bazowych, np.: primary, secondary, accent, warning, danger, surface, ink,
  • Następnie dodaj odcienie i tła do kolorów bazowych
  • Jeden bazowy zestaw zmiennych staje się podstawą do kolejnych motywów
:root {
--surface: #fff8f1;
--ink: #223047;
--accent: #ec5a2f;
--accent-soft: color-mix(in srgb, var(--accent) 20%, white);
}
.theme-green {
--surface: #f0f8f4;
--ink-local: #1a3d2d;
--accent: #17906f;
}

7

Deklaracja zmiennej z typem - @property

  • Możesz zadeklarować zmienną CSS z określonym typem i wartościami domyślnymi za pomocą @property.
  • To pozwala na animowanie zmiennych i zapewnia lepszą kontrolę nad ich typem.
  • Podstawowa składnia:
@property --my-color {
syntax: '&;lt;color>';
inherits: true;
initial-value: #ff6600;
}
@property --my-size {
syntax: '&;lt;length-percentage>';
inherits: false;
initial-value: 1rem;
}
  • syntax: określa typ wartości (np. <color>, <length>, <number>, <percentage>, <length-percentage>, <angle>, <image>, <integer>, <custom-ident>).
  • inherits: czy zmienna dziedziczy wartość z rodzica.
  • initial-value: wartość domyślna.

Kliknij przycisk, aby zobaczyć animację zmiennych zadeklarowanych przez @property.
